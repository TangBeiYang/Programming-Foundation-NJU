# C++程序设计简答题库

## 1. 冯·诺伊曼体系结构是什么？
**回答：**
冯·诺伊曼体系结构（又称存储程序式计算机）由五个基本部件组成：
1. **存储器**：用于存储程序（指令序列）和数据。
2. **运算器**：执行算术和逻辑运算。
3. **控制器**：控制程序执行，向其他部件发送控制信号。
4. **输入设备**：将外部信息输入计算机。
5. **输出设备**：将处理结果输出到外部。

*其工作过程为：程序装入存储器后，控制器逐条取出并执行指令，运算器完成运算，数据通过输入设备或存储器获取，中间结果存于存储器，最终结果通过输出设备输出。其本质是通过不断改变存储器中的数据（程序状态）来实现计算。*

## 2. 运算符的优先级是什么？
**回答：**
C++ 操作符的优先级规定了表达式中相邻操作符的执行顺序。优先级数字越小，优先级越高。基本规律如下（从高到低）：
1. 域解析符 `::`、后置自增/自减、函数调用、下标访问等。
2. 前置自增/自减、逻辑非、按位取反、取负、取正、取地址、间接访问、`sizeof`、`new`、`delete`、强制类型转换。
3. 成员指针访问符 `.*`、`->*`。
4. 乘、除、取余 `*`、`/`、`%`。
5. 加、减 `+`、`-`。
6. 移位 `<<`、`>>`。
7. 关系比较 `<`、`<=`、`>`、`>=`。
8. 相等比较 `==`、`!=`。
9. 按位与 `&`。
10. 按位异或 `^`。
11. 按位或 `|`。
12. 逻辑与 `&&`。
13. 逻辑或 `||`。
14. 条件操作符 `?:`。
15. 赋值操作符 `=`、`+=`、`-=` 等。
16. 逗号操作符 `,`。

*若优先级相同，则按结合性（左结合或右结合）决定计算顺序。*

## 3. 程序设计范式有哪些？特点是什么？
**回答：**
典型的程序设计范式有：
1. **过程式程序设计**：
   * 以功能为中心，基于功能分解和过程抽象。
   * 程序由一系列子程序（函数）构成，数据独立于子程序，通过参数或全局变量传递。
   * 适合大多数传统应用，与冯·诺依曼计算模型直接对应。

2. **面向对象程序设计**：
   * 以数据为中心，基于数据抽象和封装。
   * 程序由对象构成，对象包含数据及其操作，通过消息传递实现交互。
   * 支持继承、多态，适合大型、复用性要求高的系统。

3. **函数式程序设计**：
   * 围绕函数进行，函数可作为值传递。
   * 强调“做什么”而非“如何做”，基于递归函数理论和 lambda 演算。
   * 适合符号处理、人工智能等领域。

4. **逻辑式程序设计**：
   * 基于事实和推理规则，通过逻辑推理实施计算。
   * 强调描述性，也属于声明式编程。
   * 适合知识表示、专家系统等。

*其中，过程式和面向对象范式是目前主流，适用于大多数实际应用。*

## 4. C++ 基本数据类型有哪些？
**回答：**
C++ 基本数据类型（内置类型）包括：
1. **整数类型**：`int`、`short`、`long`（及其无符号版本 `unsigned`）。
2. **实数类型（浮点型）**：`float`、`double`、`long double`。
3. **字符类型**：`char`（单字节字符）、`wchar_t`（宽字符）。
4. **逻辑类型（布尔型）**：`bool`，取值 `true` 或 `false`。
5. **空值类型**：`void`，用于无返回值函数或通用指针。

*整数和实数类型统称算术型，整数类型、字符类型和布尔类型统称整型。*

## 5. C++ 构造数据类型有哪些？
**回答：**
C++ 构造数据类型是通过已有类型构造出的复合类型，包括：
1. **数组类型**：由多个同类型元素构成，如一维数组、二维数组。
2. **结构类型**：由多个不同类型成员构成，用于描述具有多个属性的实体。
3. **联合类型**：多个成员共享同一内存空间，用于表示多种类型的数据。
4. **枚举类型**：用户自定义的值集合，提高可读性。
5. **指针类型**：存储内存地址，用于间接访问数据。
6. **引用类型**：变量的别名，常用于参数传递。

## 6. 数据结构是什么？与数据类型有何区别？
**回答：**
*数据结构是计算机中组织、存储和管理数据的方式，关注数据元素之间的关系及操作。常见数据结构包括：*
- *数组：连续存储的相同类型元素集合。*
- *链表：通过指针连接的节点序列，支持动态插入/删除。*
- *栈：后进先出（LIFO）的线性结构。*
- *队列：先进先出（FIFO）的线性结构。*
- *树：层次化结构，如二叉树、二叉搜索树等。*

*数据类型是程序设计语言中用于定义数据的分类（如 int、float），而数据结构是更抽象的逻辑和物理组织方式，数据类型是构建数据结构的基础。*

## 7. 位运算的优缺点是什么？
**回答：**
**优点**：
- 执行速度快，直接对二进制位操作，适合底层系统编程。
- 节省内存，可对多个状态位进行紧凑存储和操作。

**缺点**：
- 可读性差，难以理解和维护。
- 容易出错，尤其是涉及符号位和移位操作时。
- 移植性可能受限，不同平台对位操作的解释可能不同。

## 8. 带参数宏与普通函数的区别是什么？
**回答：**
1. **处理阶段**：宏在预处理阶段进行文本替换，函数在编译后执行。
2. **类型检查**：宏不进行参数类型检查，函数会进行严格的类型检查。
3. **代码生成**：宏每次调用都会展开代码，可能增加代码体积；函数调用生成跳转指令，代码复用。
4. **副作用**：宏可能因多次替换导致副作用（如参数被重复计算），函数参数只计算一次。
5. **调试**：宏展开后难以跟踪调试，函数调用易于调试。

---

## 9. 带参数宏相比普通函数的优缺点是什么？
**回答：**
**优点**：
- 执行效率高，无函数调用开销（无压栈、跳转、返回）。
- 提高程序运行速度，适合简单、频繁调用的小操作。
- 可生成通用代码，不依赖具体类型。

**缺点**：
- 可能导致重复计算，增加代码体积。
- 不进行参数类型检查，容易引入错误。
- 难以调试，宏展开后不易阅读。
- 括号使用复杂，容易因优先级问题出错。

## 10. 枚举类型有什么好处？
**回答：**
枚举类型的好处包括：
1. **提高程序可读性**：用有意义的标识符代替数字，使代码更易于理解。
2. **增强程序正确性**：限定变量取值范围，避免无效值。
3. **便于维护**：相关常量集中定义，修改时只需改动一处。
4. **类型检查**：编译器可进行类型检查，避免误用。
